---

This guide walks you through creating your first AI-powered chatbot application on Dify Cloud — from signing up to calling your app via API.

<Steps>
  <Step title="Create a Dify Cloud account">
    Go to [cloud.dify.ai](https://cloud.dify.ai) and sign up with your email, Google, or GitHub account.

    Once registered, you are taken to your workspace dashboard. Every new account starts on the **Sandbox** plan, which includes 200 free GPT-4 calls and lets you explore all core features without a credit card.
  </Step>

  <Step title="Configure a model provider">
    Before building an app, connect at least one LLM provider so Dify can make API calls on your behalf.

    1. In the top-right corner, click your avatar and go to **Settings**.
    2. Select **Model Provider** from the left sidebar.
    3. Choose a provider — for example, **OpenAI** — and click **Setup**.
    4. Paste your API key and click **Save**.

    <Tip>
      Dify supports dozens of providers including OpenAI, Anthropic, Azure OpenAI, Mistral, Cohere, Hugging Face, and any OpenAI-compatible endpoint. You can add multiple providers and mix models across your apps.
    </Tip>

    After saving, the provider card shows a green checkmark and lists the available models (e.g., `gpt-4o`, `gpt-3.5-turbo`).
  </Step>

  <Step title="Create a Chatbot application">
    1. From the workspace dashboard, click **Create App**.
    2. Select **Chatbot** as the application type.
    3. Give the app a name such as `My First Chatbot` and click **Create**.

    You are taken to the **Prompt IDE** — Dify's visual interface for configuring your app's system prompt, model, and parameters.

    In the **System Prompt** field, enter an instruction for the assistant. For example:

    ```text
    You are a helpful assistant. Answer questions clearly and concisely.
    ```

    Under **Model**, select the model you configured in the previous step (e.g., `gpt-4o`).

    Click **Publish** in the top-right corner to save your configuration.
  </Step>

  <Step title="Create a Knowledge Base and attach it">
    A Knowledge Base (also called a Dataset) lets your chatbot answer questions grounded in your own documents.

    #### Create the knowledge base

    1. Click **Knowledge** in the left navigation bar.
    2. Click **Create Knowledge**.
    3. Upload a document — a PDF, Markdown file, or plain text file works well for testing.
    4. Leave the indexing settings at their defaults and click **Save & Process**.

    Dify chunks the document, generates embeddings, and stores them in the vector store. When the status shows **Available**, the knowledge base is ready.

    #### Attach it to your app

    1. Go back to your chatbot (**Studio** → select your app).
    2. In the **Context** section of the Prompt IDE, click **Add**.
    3. Select the knowledge base you just created and click **Add**.
    4. Click **Publish** to update the app.

    <Info>
      Dify uses retrieval-augmented generation (RAG) to find the most relevant chunks from your knowledge base and inject them into the prompt context before calling the LLM.
    </Info>
  </Step>

  <Step title="Test your app in the web interface">
    Click **Preview** (or **Run**) in the top-right corner of the Prompt IDE. A chat window opens.

    Type a question related to the document you uploaded and verify that the response is grounded in your content. You can iterate on the system prompt or model settings, then click **Publish** again to apply changes.

    When you're satisfied, click the **Share** button to get a public URL for your app's web interface. Anyone with the link can use the chat interface without needing a Dify account.
  </Step>

  <Step title="Call your app via the API">
    Every Dify app exposes a REST API so you can integrate it into your own product.

    #### Get your API key

    1. Open your app and click **API Access** in the left sidebar (or the top navigation).
    2. Click **API Key** and copy the key shown.

    #### Send a message

    Use the `/chat-messages` endpoint to send a message and receive a streaming response:

    ```bash
    curl -X POST 'https://api.dify.ai/v1/chat-messages' \
      -H 'Authorization: Bearer YOUR_APP_API_KEY' \
      -H 'Content-Type: application/json' \
      -d '{
        "inputs": {},
        "query": "What does the uploaded document cover?",
        "response_mode": "blocking",
        "conversation_id": "",
        "user": "user-001"
      }'
    ```

    Replace `YOUR_APP_API_KEY` with the key you copied. A successful response looks like:

    ```json
    {
      "event": "message",
      "message_id": "9da23599-e713-473b-982c-4328d4f5c78a",
      "conversation_id": "45701982-8118-4bc5-8e9b-64562b4555f2",
      "mode": "chat",
      "answer": "The document covers...",
      "metadata": {},
      "created_at": 1705407629
    }
    ```

    <Tip>
      Set `"response_mode": "streaming"` to receive server-sent events (SSE) for a typewriter-style output in real time.
    </Tip>

    To continue the conversation, pass the returned `conversation_id` in your next request.
  </Step>
</Steps>

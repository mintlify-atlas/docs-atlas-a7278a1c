---
title: Send Message
description: Send a user message in a chat conversation and receive a model response.
---

<Note>
This endpoint is for **Chat**, **Agent Chat**, and **Advanced Chat** application types only.
</Note>

## Endpoint

```
POST /chat-messages
```

## Request

<ParamField body="query" type="string" required>
  The user's message text.
</ParamField>

<ParamField body="inputs" type="object" required>
  Key-value pairs of variable inputs defined in the app's input form. Pass an empty object `{}` if there are no variables.
</ParamField>

<ParamField body="response_mode" type="string" required>
  Response delivery mode. Must be one of:
  - `blocking` — wait for the full response before returning.
  - `streaming` — stream the response using Server-Sent Events (SSE).
</ParamField>

<ParamField body="user" type="string" required>
  A unique identifier for the end user. Used for conversation tracking and rate limiting. The value is defined by your application.
</ParamField>

<ParamField body="conversation_id" type="string">
  The UUID of an existing conversation. Omit to start a new conversation.
</ParamField>

<ParamField body="files" type="array">
  List of files to attach to the message. Each file object supports:
  - `type` (string) — file type, e.g. `image`.
  - `transfer_method` (string) — `remote_url` or `local_file`.
  - `url` (string) — URL of the file (when `transfer_method` is `remote_url`).
  - `upload_file_id` (string) — ID returned from the file upload endpoint (when `transfer_method` is `local_file`).
</ParamField>

<ParamField body="auto_generate_name" type="boolean" default="true">
  Whether to automatically generate a title for the conversation from the first message.
</ParamField>

## Response

### Blocking mode

<ResponseField name="message_id" type="string">
  Unique ID of the message.
</ResponseField>

<ResponseField name="conversation_id" type="string">
  Unique ID of the conversation.
</ResponseField>

<ResponseField name="mode" type="string">
  App mode. Value: `chat`.
</ResponseField>

<ResponseField name="answer" type="string">
  The model's response text.
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional metadata including token usage and retrieval resources.
</ResponseField>

<ResponseField name="created_at" type="integer">
  Unix timestamp of message creation.
</ResponseField>

### Streaming mode

The response is an SSE stream. Each event has an `event` field:

| Event | Description |
|---|---|
| `message` | A chunk of the answer text. |
| `agent_message` | Agent mode: a chunk of the agent's response. |
| `agent_thought` | Agent mode: an intermediate reasoning step. |
| `message_file` | A file generated during the response. |
| `message_end` | Final event containing metadata and usage statistics. |
| `message_replace` | Content moderation triggered a full response replacement. |
| `error` | An error occurred during streaming. |

## Examples

<CodeGroup>

```bash Blocking
curl -X POST https://api.dify.ai/v1/chat-messages \
  -H 'Authorization: Bearer {api_key}' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "What is LangChain?",
    "inputs": {},
    "response_mode": "blocking",
    "user": "user-abc123"
  }'
```

```bash Streaming
curl -X POST https://api.dify.ai/v1/chat-messages \
  -H 'Authorization: Bearer {api_key}' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "Explain RAG in simple terms",
    "inputs": {},
    "response_mode": "streaming",
    "user": "user-abc123",
    "conversation_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6"
  }'
```

```bash With File
curl -X POST https://api.dify.ai/v1/chat-messages \
  -H 'Authorization: Bearer {api_key}' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "Summarize this document",
    "inputs": {},
    "response_mode": "blocking",
    "user": "user-abc123",
    "files": [
      {
        "type": "document",
        "transfer_method": "local_file",
        "upload_file_id": "file-id-returned-from-upload"
      }
    ]
  }'
```

</CodeGroup>

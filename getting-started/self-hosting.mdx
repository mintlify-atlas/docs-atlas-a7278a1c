Dify can be deployed on any server or cloud environment you control. Self-hosting gives you full data ownership, the ability to customise every component, and no dependency on Dify Cloud infrastructure.

## Minimum system requirements

| Resource | Minimum |
|----------|---------|
| CPU | 2 cores |
| RAM | 4 GiB |
| Disk | 20 GiB (more for large knowledge bases) |

<Warning>
  Running below these minimums results in degraded performance and may cause services to OOM-kill under load. For production workloads, 4 cores and 8 GiB RAM is a more comfortable starting point.
</Warning>

## Deployment options

<CardGroup cols={2}>
  <Card title="Docker Compose" icon="docker" href="/getting-started/docker-compose">
    The fastest way to run Dify on a single server. All services — API, worker, web frontend, PostgreSQL, Redis, Weaviate, Nginx, and the sandbox — are orchestrated by a single `docker compose up -d` command. Recommended for development, staging, and low-to-medium traffic production deployments.
  </Card>
  <Card title="Kubernetes with Helm" icon="ship" href="https://github.com/douban/charts/tree/master/charts/dify">
    For production-scale deployments that need high availability, horizontal scaling, and native Kubernetes features. Several community-maintained Helm charts are available. Suitable when you need to run multiple API replicas, separate worker pools, or integrate with managed cloud databases and object storage.
  </Card>
</CardGroup>

## Architecture overview

A standard Docker Compose deployment runs the following services:

| Service | Image | Role |
|---------|-------|------|
| `api` | `langgenius/dify-api` | Flask REST API server (Gunicorn + gevent) |
| `worker` | `langgenius/dify-api` | Celery worker — processes dataset indexing, workflows, emails |
| `worker_beat` | `langgenius/dify-api` | Celery beat scheduler for periodic tasks |
| `web` | `langgenius/dify-web` | Next.js frontend console and app runtime |
| `db_postgres` | `postgres:15-alpine` | Primary relational database |
| `redis` | `redis:6-alpine` | Cache, pub/sub, and Celery broker |
| `weaviate` | `semitechnologies/weaviate` | Default vector store for knowledge bases |
| `nginx` | `nginx` | Reverse proxy, SSL termination, static file serving |
| `sandbox` | `langgenius/dify-sandbox` | Sandboxed Python/JavaScript code execution |
| `ssrf_proxy` | `ubuntu/squid` | Outbound proxy to mitigate SSRF risks |
| `plugin_daemon` | `langgenius/dify-plugin-daemon` | Plugin lifecycle and execution manager |

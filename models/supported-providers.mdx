Dify integrates with a broad ecosystem of model providers out of the box. Providers are split into two categories: **cloud/hosted** providers that require an API key, and **local/open-source** providers that you run in your own infrastructure.

## Cloud Providers

### OpenAI

The most widely used provider in Dify deployments. Supports all model types.

| Model | Type | Notes |
|-------|------|-------|
| gpt-4o | LLM | Vision + tool-calling |
| gpt-4o-mini | LLM | Cost-efficient |
| gpt-4-turbo | LLM | 128k context |
| gpt-4 | LLM | — |
| gpt-3.5-turbo | LLM | Fastest, lowest cost |
| text-embedding-3-large | Text Embedding | Best embedding quality |
| text-embedding-3-small | Text Embedding | — |
| text-embedding-ada-002 | Text Embedding | Legacy |
| tts-1 / tts-1-hd | Text-to-Speech | Multiple voices |
| whisper-1 | Speech-to-Text | — |
| dall-e-3 / dall-e-2 | Image Generation | — |
| omni-moderation-latest | Moderation | — |

**Required credentials:** `api_key`. Optionally set `organization_id` for org-scoped usage.

## Local / Open-Source Providers

### Ollama

Run any GGUF or Safetensors model locally. The most popular option for private deployments.

| Model Type | Notes |
|------------|-------|
| LLM | llama3, mistral, gemma, phi3, qwen2, and many others |
| Text Embedding | nomic-embed-text, mxbai-embed-large |

**Required credentials:** `base_url` — default is `http://localhost:11434`.

<Tip>
For Docker-based Dify deployments accessing a host-installed Ollama, use `http://host.docker.internal:11434` as the base URL.
</Tip>

---

### LM Studio

LM Studio exposes an OpenAI-compatible API on port 1234 by default. Add it as a custom OpenAI-compatible provider.

| Model Type | Notes |
|------------|-------|
| LLM | Any model loaded in LM Studio |
| Text Embedding | Any embedding model loaded in LM Studio |

**Base URL:** `http://localhost:1234/v1`

---

### LocalAI

A drop-in OpenAI-compatible server for local model inference.

**Base URL:** `http://localhost:8080/v1`

---

### vLLM

High-throughput inference server. Expose via OpenAI-compatible mode.

```bash
python -m vllm.entrypoints.openai.api_server \
  --model meta-llama/Meta-Llama-3-8B-Instruct \
  --port 8000
```

**Base URL:** `http://localhost:8000/v1`

---

### Xinference

Distributed model serving platform supporting LLMs, embeddings, and rerankers.

| Model Type | Supported |
|------------|----------|
| LLM | Yes |
| Text Embedding | Yes |
| Rerank | Yes |
| Image Generation | Yes |

**Required credentials:** `server_url`, `model_uid`.
